### 머신러닝이란 무엇인가
 - 머신러닝은 데이터를 기반으로, 패턴을 학습하고 그 패턴을 바탕으로 새로운 데이터에 대해서 결과를 예측하는 기술을 말 합니다.
 - 프로그래머는 규칙을 찾는 것이 아니라, 규칙을 찾을 수 있는 수식과 학습 방식을 제공하여 컴퓨터가 제공된 데이터를 확인하면서 패턴을 찾습니다.

### 왜 머신러닝을 하는가?
 - 과거 한 때 전문가적 방식 (if-else 를 거의 도배 하는 방식) 으로 한 적이 있었는데
   - 너무 많은 예외가 존재하며, 예외를 모두 기술 할 수도 없고, 선형으로 분리 가능하지 않은 경우도 많이 있는데다가
   - 그런 방식으로 나눌 수 없는 경우, 기계가 무엇을 생성해야 하는 경우에는 전혀 쓸 수가 없었기 때문에
   - 사람이 직접 규칙을 작성하는 방식으로는 규칙 기반으로 나누기 너무 어렵고 그런 방식으로 생성은 거의 할 수 없었기 때문입니다.
- 그렇기 때문에 우리는 기계가 스스로 방대한 데이터에서 규칙을 찾기를 기대하면서 머신러닝을 합니다.

### 일반화 (Generalization)
 - 머신러닝의 목표는 본 적 없는 데이터를 잘 예측하는 것 입니다.
   - 생성형 모델도 결국 주어진 데이터에서, 다음 나올 내용을 예측하는 문제로 바꿔 쓸 수 있습니다.
   - 생성 모델도 본질적으로는 분류/회귀 문제와 동일합니다. 그 규모가 아주아주아주 클 뿐이죠.
     - 개념적으로는 어떤 문장 / 그림이 들어왔을 때 다음 나올 문장 / 그림으로 적당한 것은? 을 아주 많이 풀고있는 것 입니다.
   - 그렇기 때문에 본 적 없는 새로운 데이터를 잘 예측하는 것이 진짜 중요한데요
     - 훈련 데이터만 잘 맞추는 모델을 과적합 (overfitting) 되었다고 합니다. -> 이렇게 되면 모델은 쓰레기입니다.
     - 새로운 데이터를 잘 맞추는 모델 -> 일반화된 모델.
   - 그래서 현대 인공지능은 또 overfitting 을 없애는 방법도 여러가지 사용하고 있습니다.
     - 물론 overfitting 에 대해서도 여러 견해가 있지만, 현재 다룰 내용은 아니기 때문에 위 내용정도만 알아도 충분 합니다.
   - 그래서 단순히 데이터를 "기억" 하는것이 아니라.
     - 해당 데이터들에서 패턴을 찾아서, 공통 규칙을 찾아내는것이 굉장히 중요합니다.

### 머신러닝의 분류
  - 머신러닝은 크게 3 가지로 나눌 수 있습니다.
    - 지도학습
    - 비지도학습
    - 강화학습


### 지도학습
 - 입력과 정답을 모두 제공합니다.
 - 정답을 예측하는 모델을 학습합니다.
 - 가장 널리 사용되는 방식입니다.
 - ex)
   - 개와 고양이 분류
   - 집 크기 / 위치 -> 가격 예측 (회귀 문제)
 - 지도학습은 데이터에 정답이 필요합니다.
 - 이러한 정답을 label 이라고 하며 label된 데이터를 구하기 어려운 경우들이 종종 있습니다.

### 비지도학습
 - 정답이 없습니다 (label 이 없음)
 - 데이터의 구조를 발견하는 학습
 - 군집 / 압축 / 분포 등

 ex)
  - 비슷한 고객분류 (K-Means Algorithm)
  - 차원 축소 및 압축 (Principal Component Analysis, Auto Encoder)

### 강화학습
 - 평가가 가능한 (정해진 정답은 없지만, 어떤 행동이 어떤 결과를 가지고온다는 계산 가능하면 됨) 환경에서 어떤 시도 (action) 를 통해 보상 (Reward) 을 받아서 학습
 - 의사결정 문제에 강력함을 보여줌
   - 바둑등의 보드게임
   - 전략 시뮬레이션 게임 등등...
 - 정답은 없고, 대신 보상이 있음. 이 보상이 최대인지 아닌지는 복잡한 문제가 되면 인간조차 모름.
   - ex) 바둑의 최적해를 인간이 알고 보상 모델을 설계하는것이 아니기 때문


### 머신러닝으로 해결할 수 있는 문제
 - 회귀 (Regression)
   - 연속적인 숫자를 예측
     - 집값 예측, 주식 예측 등등...
 - 분류 (Classification)
   - 카테고리를 예측합니다.
     - 남/녀, 개/고양이 등등...
 - 군집
   - 정답 없이 데이터들을 비슷하다고 묶기만 합니다.
     - 고객 카테고리화, 문서 카테고리화
 - 차원 축소
   - 데이터를 중요한 특징 몇 가지로 압축합니다.
 - 생성
   - 새로운 데이터를 생성합니다.
     - 이쪽은 GAN을 제외하고는 거의 대부분 Classification + Regression 이기도 합니다.


### 머신러닝의 업무 파이프라인
 - 문제 정의
   - 나는 지금 어떤 문제를 풀고 싶은 것인가?
 - 데이터 수집
   - 이러한 문제를 풀기 위해서 관련된 데이터는 무엇이 있는가.
 - 데이터 전처리 (학습 가능하게 format 맞추기, 쓰레기 걸러내기)
   - 정규화, 정답 수집, 토큰으로 분리, 노이즈 제거 등등...
 - 데이터 분리 (Train Dataset / Test Dataset)
 - 적절한 모델 선택
 - 학습 (Train)
 - 평가 (Evaluation)
 - 배포 (Deployment)
 - 피드백

위와 같은 과정을 거치면서 최적의 모델을 만듭니다.
또한 자기가 만든 모델의 긴급한 서비스 CS에 대해서도 응대할 수 있어야합니다.
이런것들을 하려면 모델 앞뒤로도 많은것들이 필요하지만, 우선은 이런 내용은 차차 알게 될 것이니 지금은 몰라도 됩니다.


### 머신러닝과 딥러닝의 관계
 - 인공지능 내에 여러 분류가 있습니다.
  - 머신러닝은 인공지능의 한 갈래이며
  - 딥러닝은 머신러닝의 한 갈래입니다.
  - 신경망을 많이 쌓아 이미지/언어같은 고차원 데이터를 잘 분석하려고 시도하는것이 딥러닝 입니다.
  - 물론 강화학습에도 역시 딥러닝을 사용할 수 있습니다.
