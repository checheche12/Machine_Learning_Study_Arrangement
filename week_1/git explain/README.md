### Git 소개

- Git은 리눅스를 만든 리누스 토발즈가, 자신의 리눅스 코드를 관리하기 위해 만든 버전 관리 시스템 입니다.
- 코드를 작성하는 사람이라면 Git 은 아니더라도 버전 관리 는 꼭 해주는게 좋은데요
- Git 이 버전 관리 도구중 가장 유명하고 많이 사용하기 때문에, 그리고 많은 기업체나 연구에서도 사용하기 때문에 꼭 사용할 수 있으면 좋은 도구입니다.



### Git 이란?
- 파일의 변화를 변경 사항에 따라 기록하고, 되돌리거나 병합하는 등의 작업을 할 수 있게 해주는 도구입니다.
- 문서의 히스토리를 관리 할 수 있습니다.
- 해당 자료도 전부 git으로 올라가고 있습니다.


| 문제 상황            | Git이 없다면          | Git이 있다면           |
| ---------------- | ----------------- | ------------------ |
| 코드를 수정했는데 오류 발생  | 원본을 복구하기 어렵다      | 이전 커밋으로 손쉽게 복원     |
| 여러 명이 동시에 코드를 수정 | 파일 충돌, 덮어쓰기       | 브랜치를 나눠 안전하게 협업    |
| 새로운 시도를 해보고 싶음   | 기존 코드 망가질 위험      | 별도 브랜치에서 실험 후 병합   |
| 코드 히스토리 추적       | 이게 왜 이렇게 바뀌었더라? | 커밋 로그로 이유/시점 추적 가능 (다만 로그를 깔끔하게 작성 해야 합니다.)|


### 어디에 사용하는가
 - 개인 프로젝트 관리
 - 협업 개발
 - 오픈소스 생태계
 등등... 사실 개발 전반적으로 많이 사용하는 도구입니다.


Git vs GitHub
| 구분         | 설명                                           |
| ---------- | -------------------------------------------- |
| Git    | 로컬에서 버전을 관리하는 도구 (버전관리 기술 자체)                |
| GitHub | Git 저장소를 온라인으로 공유/협업할 수 있는 플랫폼           |
| 비유하자면… | Git은 도구, GitHub = 그 도구로 만든 작품을 모아두는 플랫폼 |


그래서 GitHub 말고도 GitLab 과 같은 플랫폼들도 존재합니다.
그렇기 때문에 각 회사들별로 자신들의 Git 저장소를 가질 수 있습니다.


### 왜 사용하는가?
- 내가 무엇을 왜 바꾸었는지 기록이 됨.
- 브랜치 별 작업을 통해 협업이 표준화 됨
- 사실상 현대적 개발의 표준점


### 여러가지 기능이 있는데
- 저장소 초기화부터, PR Merge, Rebase 까지는 2 주차에 python 코드를 실습 하면서 진행 해볼 예정입니다.
