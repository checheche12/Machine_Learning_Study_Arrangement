## 1주차, Data Structure


### Stack (스택)
 - 롤에서 나서스 스택 쌓는다.
 - 여친 분노 스택 폭발 했다.
 등등에서 우리가 실 생활에서 많이 사용하는 단어 "스택" 입니다.
 의미도 동일하고, 의미는 "쌓는다" 라는 의미를 가집니다.

 쌓는다 라는 의미를 가진 만큼, 쌓여있는 것을 꺼내기 위해서는 먼저 들어간 것이 나중에 나올 수 밖에 없는 구조입니다.

 이러한 구조를 후입선출 (Last In, First Out) 구조라고 합니다.


```python3
stack = []
stack.append(10)      # push
top = stack.pop()     # pop
```


 ### Queue (큐)
 - 롤 큐 돌린다.
 - 오버워치 큐 잡는중
 등으로 많이 사용하는 단어 큐 입니다.
 대기열 이라는 의미가 있으며, 들어온 순서대로 나오는 구조로 되어있습니다.

 이러한 구조를 선입 선출 (First In, First Out) 이라고 합니다.


```python3
from collections import deque
q = deque()
q.append(1)          # enqueue
x = q.popleft()      # dequeue
```

### Hash Table
 - 우선 해시라는 것에 대해서 이해해야 하는데
 - 해시 브라운 처럼, 무언가를 으깨버리는 느낌입니다.
 - 이렇게 으깨는 도구를 해시 함수라고 하고, 어떤 데이터를 해시 함수에 넣어서, 테이블에 저장합니다.
 - 이러한 구조를 해시 테이블이라고 합니다.


```python3
counter = {}
for x in [1,2,1,3,2,1]:
    counter[x] = counter.get(x, 0) + 1
# {1: 3, 2: 2, 3: 1}
```


### Tree (트리)
- 사이클 (순환) 이 없는 연결 그래프
- 루트에서 시작해서 계층 구조를 표현합니다.
- 가장 중요한것은 Binary Tree 인데, 이것은 한 계층에서 다음 계층으로의 가지가 0~2개 를 가집니다. 3개 이상을 가지지 않습니다.

```python3
import heapq
nums = [5, 1, 7, 3]
heapq.heapify(nums)        # O(n)
heapq.heappush(nums, 2)    # O(log n)
m = heapq.heappop(nums)    # 최솟값
```


| 구조         | 핵심 특성    | 주요 연산           | 평균 시간복잡도          | 대표 용도          |
| ---------- | -------- | --------------- | ----------------- | -------------------- |
| Stack      | LIFO     | push/pop/top    | O(1)              | Undo, 괄호검사, DFS |
| Queue      | FIFO     | enqueue/dequeue | O(1)              | BFS, 버퍼        |
| Hash Table | 키→인덱스 매핑 | put/get/remove  | O(1) (최악 O(n))    | 카운팅, 캐시, 심볼 테이블      |
| Tree       | 계층/정렬/범위 | 탐색/삽입/삭제        | (BST) 평균 O(log n) | 폴더·UI, 자동완성, 힙/세그트리  |


시간 복잡도(Time Complexity) 는 알고리즘의 효율성을 나타내는 척도로,
입력 크기 n이 커질 때 연산 횟수가 얼마나 빠르게 증가하는가를 수학적으로 표현한 것입니다.
일반적으로 Big-O 표기법(O-notation) 을 사용합니다.

예를 들어:
 - O(1): 입력 크기와 상관없이 일정한 시간 (ex. 배열 인덱스 접근)
 - O(n): 입력 크기에 비례 (ex. 전체 탐색)
 - O(log n): 입력이 커질수록 느리게 증가 (ex. 이진 탐색)
 - O(n²): 중첩 반복문 구조 (ex. 버블 정렬)
 - O(2ⁿ): 지수시간 (ex. 부분집합 탐색) 매우 빠르게 증가
 - O(n!): 팩토리얼 시간 (ex. 순열 생성, 외판원 문제), 입력이 조금만 커져도 현대 컴퓨터로 거의 계산 불가능.
 ... 이런식으로 안에 들어간 함수의 차수가 클 수록 느려진다고 보면 됩니다.
